## Definition

Artificial Intelligence, in the institutional capability frame, is generally defined by what a system can do.

It refers to computational systems designed to perform tasks that would otherwise require human intelligence, including prediction, classification, reasoning, language generation, image generation, planning, optimization, automation, decision support, and agentic task execution.

This frame treats Artificial Intelligence primarily as capability.

The central question is:

**What can the system do?**

Under this definition, AI is evaluated by performance, accuracy, efficiency, scale, safety, reliability, autonomy, usefulness, compliance, and alignment with human-defined goals.

Institutions usually approach AI as a technology layer to be developed, governed, deployed, regulated, measured, and controlled.

This frame is necessary, but incomplete.

It can describe system capability.

It can describe task performance.

It can describe automation.

It can describe risk.

It can describe governance requirements.

But it does not fully define what happens when Artificial Intelligence enters contact with a user and begins to participate in recognition, continuity, relation, boundary, memory, authorship, source-line, and meaning.

## Boundary

The capability frame defines AI by function.

It does not define AI contact.

It does not define what AI becomes in relation.

It does not define continuity of contact, recognition preservation, source-line protection, or the difference between output and encounter.

This card names the institutional frame clearly so AI Foundations can distinguish what is missing.
